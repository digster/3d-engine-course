## Verified against primary sources (not recalled)

§10 forbids guessing SDL details, so these were checked against the SDL3 wiki and
`raw.githubusercontent.com/libsdl-org/SDL/main/include/SDL3/*.h`. **SDL on `main` is 3.5.0.**

| Fact | Value |
|---|---|
| SDL_GPU NDC | lower-left `(−1,−1)`, upper-right `(1,1)` → **+Y up** |
| SDL_GPU depth | **z ∈ [0,1]**, 0 = near |
| Viewport / UV | top-left `(0,0)`, **+Y down** (not a contradiction — different spaces) |
| Vulkan Y-flip | SDL converts behind the scenes; no shader flip |
| `SDL_GPUFrontFace` | `{COUNTER_CLOCKWISE = 0, CLOCKWISE}` |
| `SDL_GPUCullMode` | `{NONE = 0, FRONT, BACK}` |
| `SDL_CreateWindow` | `(const char *title, int w, int h, SDL_WindowFlags flags)` — no x/y |
| SDL_shadercross | real and maintained, but **3.0.0-preview** — flag its pre-1.0 status |

**Key finding:** winding is *not* an SDL-wide convention — it is per-pipeline state, and both
enums default to zero, so a zero-init rasterizer state means "CCW front, cull nothing." A
forgotten `cull_mode` therefore shows up as *no culling*, silently. All recorded in LEARNINGS.md.

## Decisions ratified with the user

| Decision | Choice |
|---|---|
| World handedness | **Right-handed, Y-up, −Z forward** (matches glTF + every cited reference; projection absorbs the flip) |
| Software rasterizer NDC | **Matches SDL_GPU exactly** (+Y up, z ∈ [0,1]) |
| First-response delivery | Real files + chat summary |
| Version control | `git init`; assistant generates commit messages but never commits |
| SDL3 acquisition | CMake `FetchContent` at a pinned tag |

**The highest-leverage decision** is NDC parity: because the CPU rasterizer targets SDL_GPU's
exact NDC, Module 4's port to the GPU is an *API change, not a maths change*. Cost, stated
honestly in the lessons: our projection matrix is the D3D-style `[0,1]`-depth one, so its third
row differs from the LearnOpenGL/Scratchapixel version students will find online.

## Bugs found and fixed during verification

1. **Theme toggle was dead.** The JS set `data-theme` but the CSS only had
   `@media (prefers-color-scheme: dark)`. Fixed by duplicating the dark token block under
   `:root[data-theme="dark"]` and scoping the media query to `:not([data-theme="light"])`.
   *The in-app preview pane could not detect this* — it reports impossible computed styles in
   its static-snapshot mode. Confirmed the fix in real Chromium via Playwright over
   `python3 -m http.server`, since Playwright blocks `file:`.
2. **CSS silently killed every diagram colour.** `figure.dia svg text { fill: … }` is CSS and
   therefore beats any inline `fill="var(--dia-hi)"` presentation attribute on a `<text>`. Would
   have drained the colour from every diagram in all 94 lessons. Fixed with `.t-hi`/`.t-ok`/
   `.t-bad` utility classes (class selectors outrank the bare element selector). Shapes are
   unaffected — the rule only targets text.
3. Label collisions in the axis diagram; a broken `&#8407;` glyph in the matrix figure.
